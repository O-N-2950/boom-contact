## Principes de rédaction respectés

- Wording prudent : « dossier numérique structuré et horodaté »,
  « destiné à faciliter la déclaration », « à transmettre à votre assureur ».
- Aucune affirmation de « validité légale mondiale », « certification »,
  « reconnaissance par toutes les assurances » sans validation juridique.
- boom.contact ne détermine pas les responsabilités, ne remplace pas
  les secours, la police, un avocat ou un assureur — rappelé partout.
- Couverture explicite : micro, photos, OCR, localisation, signature,
  paiement, IA/transcription, sous-traitants, conservation, suppression,
  analytics, stores Apple/Google.

## Prochaine étape

Transmettre l'ensemble à un juriste suisse/européen pour validation et
adaptation finale. Une fois validés, ces textes alimenteront :

- les pages publiques (CGU, Privacy) du site et de l'application ;
- les fiches App Store (App Privacy) et Google Play (Data Safety) ;
- les communications externes ;
- les contrats avec les sous-traitants (DPA).

Le verrouillage juridique est l'un des deux derniers bloqueurs avant
publication stores publique, avec la validation runtime native sur
appareils.
